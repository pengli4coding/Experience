为什么要使用泛型？

先看一下下面这段代码

package com.pl.test;

import java.util.ArrayList;
import java.util.List;

@SuppressWarnings("unchecked")
public class TestGeneric {

	public static void main(String[] args) {
		List list = new ArrayList();// 可以加入Object类型的数据
		list.add("aa");// 加入String类型的数据
		list.add("bb");// 加入String类型的数据
		list.add(123);// 加入int类型的数据
		for (int i = 0; i < list.size(); i++) {
			// 要使用String类型的特性和方法，必须先进行类型转换，而这一步一不小心就会报错，因为list中会有非String类型的数据
			String item = (String) list.get(i);
			System.out.println(item);
		}
	}
}

运行结果如下，报错了哈哈。。

aa
bb
Exception in thread "main" java.lang.ClassCastException: java.lang.Integer cannot be cast to java.lang.String
	at com.pl.test.TestGeneric.main(TestGeneric.java:15)

从上面的代码中我们发现如下的两个问题：

1、当我们将一个对象放入集合中，集合不会记住此对象的类型，当再次从集合中取出此对象时，改对象的编译类型变成了Object类型，但其运行时类型任然为其本身类型。

2、取出集合元素时需要人为的强制类型转化到具体的目标类型，且很容易出现“java.lang.ClassCastException”异常。

因此，我们今天要学习的泛型就应运而生了




泛型可以运用在接口、类、方法上面




泛型类

最基本的泛型类的定义如下：

package com.pl.test;

public class Box<T> {

	private T data;
	
	public Box(T data) {
		this.data = data;
	}

	public T getData() {
		return data;
	}

	public void setData(T data) {
		this.data = data;
	}
	
}

使用如下：

package com.pl.test;

public class TestGeneric {

	public static void main(String[] args) {
		Box<Integer> box=new Box<Integer>(123);
		System.out.println(box.getData());
		box.setData(321);
		System.out.println(box.getData());
	}
}
