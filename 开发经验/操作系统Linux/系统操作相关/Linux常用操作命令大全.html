man 命令  显示命令的帮助手册
命令 --help  显示命令的详细帮助信息

文件操作相关命令

cd /etc  切换到/etc目录
cd ..  切换到上一级目录
cd test  切换到当前目录的test文件夹下
pwd  显示当前目录的全路径名
clear  清屏
ls  列出当前目录下的所有文件（不显示隐藏文件，不显示详细信息）
ls -al  列出当前目录下的所有文件（显示隐藏文件，显示详细信息）,-a表示显示包括隐藏文件在内的所有文件，-l表示显示文件的详细信息
ls -al test  列出当前目录的test目录下的所有文件（显示隐藏文件，显示详细信息）,-a表示显示包括隐藏文件在内的所有文件，-l表示显示文件的详细信息
ls -al /etc  列出/etc目录下的所有文件（显示隐藏文件，显示详细信息）,-a表示显示包括隐藏文件在内的所有文件，-l表示显示文件的详细信息
touch test.java  在当前目录下创建一个名为test.java的空文件
mkdir test  在当前目录下创建一个名为test的目录
mkdir -p d1/d2/d3  在当前目录下创建一个名为d1的目录,接着在d1目录下创建d2目录，接着在d2目录下创建d3目录
cp test.java test  拷贝当前目录下的test.java文件到当前目录下的test文件夹下面
cp -r /etc /home/test  拷贝/etc目录到/home/test目录下，-r表示拷贝的是目录
cp -r test test1  拷贝当前目录下的test文件夹到当前目录下的test1文件夹下面（注意一点的是，如果当前目录下没有test1文件夹，则会创建一个test1文件夹，然后将test文件夹下的所有文件/文件夹拷贝到test1文件夹下面）
mv test.java test  剪切当前目录下的test.java文件到当前目录下的test文件夹下面
mv test1 test  剪切当前目录下的test1文件夹到当前目录下的test文件夹下面
mv /home/test.java /home/test  剪切/home/test.java文件到/home/test文件夹下面
mv test.java test1.java  修改当前目录下的test.java文件的文件名为test1.java
rm Test.java  删除当前目录下的Test.java文件（系统会弹出确认信息）
rm -f Test.java  删除当前目录下的Test.java文件（强制删除，系统不会弹出确认信息），-f表示强制删除
rm -r test  删除当前目录下的test文件夹（系统会弹出确认信息），-r表示删除的是文件夹
rm -rf test  删除当前目录下的test文件夹（强制删除，系统不会弹出确认信息），-r表示删除的是文件夹，-f表示强制删除
rm -rf /home/test  删除/home/test文件夹（强制删除，系统不会弹出确认信息），-r表示删除的是文件夹，-f表示强制删除
cat Test.java  显示当前目录下Test.java文件的内容（cat命令的缺点是当文件内容非常长的时候，会被滚屏过去，无法看全内容）
cat /home/test/Test.java  显示/home/test/Test.java文件的内容（cat命令的缺点是当文件内容非常长的时候，会被滚屏过去，无法看全内容）
more Test.java  显示当前目录下Test.java文件的内容（more命令相比cat命令是使用分页的方式查看文件的内容，遇到内容非常长的文件的时候也可以看全，回车是翻下一行，空格是翻下一页，q是退出）
more /home/test/Test.java  显示/home/test/Test.java文件的内容（more命令相比cat命令是使用分页的方式查看文件的内容，遇到内容非常长的文件的时候也可以看全，回车是翻下一行，空格是翻下一页，q是退出）
less Test.java  分屏查看当前目录下Test.java文件的内容（less命令相比more命令会更加高效，因为less命令不会一次过先加载整个文件，而是按需加载，所以比more的效率更高）
head -10 Test.java  显示当前目录下Test.java文件的前10行内容
tail -10 Test.java  显示当前目录下Test.java文件的最后10行内容
tail -f Test.java  动态显示当前目录下Test.java文件的末尾的内容，这种用法常用于动态查看控制台文件
ln Test.java test.hard  为当前目录下的Test.java文件创建名为test.hard的硬连接（硬连接就是文件的备份，当原文件内容发生变化的时候，硬连接文件同步发生变化，原文件与硬连接文件的文件大小始终保持一致的）
ln -s Test.java test.soft  为当前目录下的Test.java文件创建名为test.soft的软连接（软连接就相等于window下的快捷方式的意思），-s表示创建软连接的意思

文件权限相关命令

chmod o+r Test.java  为当前目录下的Test.java文件赋予其他用户读权限（+代表赋予权限；u代表文件所属用户、g代表文件所属用户所在的组、o代表其他用户；r代表读权限、w代表写权限、x代表可执行权限）
chmod o-r Test.java  为当前目录下的Test.java文件剥夺其他用户读权限（-代表剥夺权限；u代表文件所属用户、g代表文件所属用户所在的组、o代表其他用户；r代表读权限、w代表写权限、x代表可执行权限）
chown xiaolili Test.java  改变当前目录下的Test.java文件的所有者为xiaolili
chgrp xiaolili Test.java  改变当前目录下的Test.java文件的所属组为xiaolili

查找相关命令

which ls  查找ls命令所在的目录
find /etc -name init  在/etc目录下（包括子目录）查找文件名为init的文件
find /etc -name *init  在/etc目录下（包括子目录）查找文件名以init为结尾的文件
find /etc -name init*  在/etc目录下（包括子目录）查找文件名以init为开头的文件
find /etc -name *init*  在/etc目录下（包括子目录）查找文件名包含init的文件
find /etc -size +1024  在/etc目录下（包括子目录）查找文件大小大于1024数据块的文件（一个数据块等于512字节）
find /etc -user root  在/etc目录下（包括子目录）查找文件所属者是root的文件
grep ftp /etc/services  对/etc/services文件内容进行查找，找出包含ftp字样的行并输出

单个文件压缩相关命令

gzip Test.java  对当前目录下的Test.java文件进行压缩操作，最终生成Test.java.gz文件，并且不保留原文件
gzip -c Test.java>Test.java.gz  对当前目录下的Test.java文件进行压缩操作，最终生成Test.java.gz文件，并且保留原文件Test.java，-c参数可以保留原文件
gunzip Test.java.gz  对当前目录下的Test.java.gz文件进行解压缩操作，最终解压出来Test.java文件（注意，最终解压出来的文件名是去掉.gz后缀后的文件名，而不是压缩文件里面包含的真实文件名），并且不保留原文件
gunzip -c test.gz>Test.java  对当前目录下的test.gz文件进行解压缩操作，最终解压到Test.java文件，并且保留原文件test.gz，-c参数可以保留原文件

多个文件（目录）打包压缩相关命令

tar -cvf test.tar test  对当前目录下的test文件夹进行打包，最终生成test.tar文件，test文件夹也保留下来，-c表示打包，-v表示显示详细信息，-f是指定文件名
tar -zcvf test.tar.gz test  对当前目录下的test文件夹进行打包并压缩操作，最终生成test.tar.gz文件，test文件夹也保留下来，-c表示产生.tar打包文件，-v表示显示详细信息，-f是指定文件名，-z表示打包同时压缩
tar -zxvf 1.tar.gz  对当前目录下的1.tar.gz文件进行解包并解压缩操作，最终解压出来etc文件（压缩的就是etc文件夹），1.tar.gz文件也保留下来，-x表示解包，-v表示显示详细信息，-f是指定要解压的文件名，-z表示打包同时压缩
tar -zxvf jdk-8u171-linux-x64.tar.gz -C /usr/local  对当前目录下的jdk压缩包进行解包并解压缩操作，最终解压到/usr/local目录下，-C表示指定解压到哪个目录下

zip压缩解压缩相关命令

zip b.zip a.txt  对当前目录下的a.txt文件进行压缩生成b.zip文件在当前目录下，并且a.txt文件也会保留下来
zip -r b.zip d1  对当前目录下的d1文件夹进行压缩生成b.zip文件在当前目录下，并且d1文件夹也会保留下来，-r表示对文件夹进行压缩
unzip b.zip  对当前目录下的b.zip进行解压缩
unzip -d /home b.zip  对当前目录下的b.zip进行解压缩到/home目录下，-d表示指定解压缩到哪个目录下

输出重定向

ls -al /etc > result.txt  把执行ls -al /etc命令的结果输出到当前目录下的result.txt文件中，>会覆盖原文件的内容，如果当前文件夹中不存在resutl.txt文件的话，则会创建result.txt文件
ls -al /etc >> result.txt  把执行ls -al /etc命令的结果追加到当前目录下的result.txt文件中，>>不会覆盖原文件的内容，会在原文件的基础上追加内容，如果当前文件夹中不存在resutl.txt文件的话，则会创建result.txt文件

管道（管道就是将一个命令的输出传送给另一个命令）

ls -l /etc | more  将ls -l /etc命令的输出传送给more命令分页显示
ls -l /etc | grep init  将ls -l /etc命令的输出传送给grep命令查找包含有init字样的行
ls -l /etc | grep init | wc -l  将ls -l /etc命令的输出传送给grep命令查找包含有init字样的行再输出到wc命令统计一共有多少包含init字样的行

网络相关命令

ifconfig eth0  查看eth0这一块网卡的具体信息
